## 五、工作流

```
采集（Capture） → 整理（Develop） → 关联（Connect） → 输出（Output）
   000_inbox          200_Card         卡片之间链接      文章/项目
                      400_Source
```

### 日常流程

1. 想法/素材先丢进 `000_inbox/`
2. 定期整理：筛选值得做成卡片的，用模板写入 `200_Card/`
3. 写卡片时建立双向链接
4. 卡片攒够后，串成文章或项目输出

## 六、核心约定

### 制卡前两问

1. 我为什么要写这个想法？
2. 我能拿它干什么？

答不上来就暂时不记。

### 三大原则

- **原子化**：一张卡片只记一个观点
- **扁平化**：不给知识预设分类，让链接自然形成结构
- **连接驱动**：卡片的价值在于它与其他卡的链接

### 双向链接

```
←  来源/引用  — "这个观点来自哪"
→  延伸/去向  — "这个观点可以延伸出什么"
```

写了 A → B，B 就应补 ← A。两头都得连上。

### 必要难度

存储越轻松，提取越困难。制卡的过程就是必要难度的体现——自己提炼、自己写，才能记住。

## 八、后续推荐方向

| 方向 | 适用时机 | 做什么 |
|------|---------|--------|
| 卡片链接与联网 | 攒到 20+ 张卡片后 | 自动发现跨领域关联、找孤立卡片 |
| 自动补双向链接 | 卡片间有引用但缺反向链 | 扫描全库检测单向引用，自动建议补全 |
| 卡片深度加工 | 觉得某张卡片没想透时 | 追问前提、找反例、延伸新维度 |
| 卡片碰撞 | 手上有两张观点矛盾的卡片 | 让它们互相辩驳，激发新洞见 |
| 内容提炼制卡 | 笔记/随记积多了 | 从已有笔记中识别有价值的原子观点制成卡片 |
| 写作输出 | 卡片攒够一个主题 | 基于卡片网络生成文章框架，串联素材成文 |
| 知识拓宽 | 想跳出当前领域 | 基于卡片主题向外推荐书、影音、人物、思路 |
| 提问打磨 | 想深入探讨某个问题 | 三阶段打磨提问，多角度分析 |
