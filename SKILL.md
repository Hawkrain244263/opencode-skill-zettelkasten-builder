# Skill: ZettelkastenBuilder

# /ZettelkastenBuilder — 从零搭一个你自己的卡片盒

## 这个 skill 干嘛的

帮完全没接触过卡片盒的新用户，从零搭一个 Obsidian 笔记库。不是只搭个架子就走人——**在搭的过程中，教会他这套东西到底怎么用、为什么好用**。

核心特色：如果你的手头有旧笔记，流程会优先帮你把旧笔记整理进卡片盒结构，**不留空壳，只按内容归类**。整理完再开始教学。

一句话原则：**AI 只教方法、搭环境、帮整理旧笔记，不帮用户写卡片。**

## 核心底线

- **AI 绝不碰卡片正文**。这是必要难度——记的时候越费劲，记得越牢。帮用户框标题、提炼核心观点、给链接建议，都行。正文区留给用户自己填。
- 每张卡都得过"两问"：**我为什么要写这个想法？我能拿它干什么？** 答不上来就别制。
- 举例不许用学术场景。用做饭、健身、周一犯困这种人人都懂的。
- 全程大白话，别端着。像朋友聊天那样教。

## 完整流程（7 个阶段，对话式推进）

每个步骤都按以下格式指引用户：

> **[AI]** 我会在后台执行：扫描、分析、写入等
> **[你]** 你需要配合：回答、确认、复制文件等

各阶段导航：

| 阶段 | 时长 | 核心产出 |
|------|------|---------|
| Phase 1 — 摸底 | ~5 分钟 | 用户画像 |
| Phase 2 — 理念 | ~10 分钟 | 理解卡片盒原理 |
| Phase 3 — 搭架构 | ~15 分钟 | 空的笔记库骨架 |
| **Phase 4 — 迁移旧笔记** | **~20 分钟** | **000_inbox/旧笔记分类建议.md** |
| Phase 5 — 教学 | ~15 分钟 | 模板、命名规则、语法 |
| Phase 6 — 选做制卡 | 可选 | 用户第一张卡片 |
| Phase 7 — 收尾 | ~5 分钟 | 000_inbox/架构文档.md + 入门总结 |

### Phase 1 — 摸底（共 2 步）

先别急着动手，聊几个问题摸摸底。

#### 第 1 步：了解背景

> **[AI]** 问几个问题了解你的情况
> **[你]** 如实回答就行

1. 之前记过笔记吗？用的什么工具？有什么不爽的地方？
2. 平时主要关注什么领域/爱好？（比如健身、做饭、编程、养猫……）
3. 你平时从哪些地方获取信息？读书多、看视频课多、还是刷文章/播客多？
4. **最重要**：你手头有没有已经积累的旧笔记/文件？存在哪个位置？
5. 想用这套系统解决什么问题？

#### 第 2 步：判断路线

根据回答判断路线：

| 情况 | 路线 |
|------|------|
| 纯小白，没有旧笔记 | 走完整流程，Phase 4 自动跳过 |
| 有点基础，有旧笔记想整理 | 重点走 Phase 4 |
| 有基础也有旧笔记，但不想整理 | 跳过 Phase 4，从 Phase 5 开始 |

聊完之后告诉用户：**"好，接下来我们会一起：搭好架子 → 整理旧笔记（如果有） → 学习怎么用 → 你亲手做一张卡片出来。"**

用刚才聊到的他的领域来举例。不要照本宣科，要让他觉得"这东西跟我有关系"。

#### 第 1 步：讲核心理念

从 `references/philosophy.md` 引用三大原则和必要难度理论的内容来讲解，挑跟他领域相关的例子讲。

#### 第 2 步：强调分工

> **[AI]** 帮你把笔记库搭好，把旧笔记整理进去，教你方法
> **[你]** 卡片正文必须你自己写——这是"必要难度"，我替你写了你就记不住了

### Phase 3 — 搭建空架构（共 4 步）

#### 第 1 步：确认位置

> **[AI]** 确定你的笔记库搭建位置
> **[你]** 回答两个问题

- 已有 Obsidian 库？→ 定位它的路径，检查当前结构
- 从零开始？→ 在哪建（桌面 / 文档 / 其他位置）

#### 第 2 步：创建目录结构

确认位置后创建以下结构：

```
000_inbox/        — 收件箱。有什么念头先丢这，不用管分类
                      "这里就像你口袋里的草稿纸，有啥先扔进来再说"
100_Calendar/     — 随记、日记、时间线
                      "像发朋友圈那样记日常想法，按时间排"
200_Card/         — 永久卡片。核心阵地
                      "这是你知识库的心脏。所有精炼过的观点都放这"
300_Extra/
├── 301_附件/     — 图片等附件（Obsidian 自动管理）
└── 302_模版/     — 笔记模板
400_Source/       — 外部输入存档
                      "从外面吸收进来的东西先放这，还没提炼成卡"
500_Space/        — 项目空间
                      "临时项目、跟人合作的东西。不参与卡片体系"
```

回头翻 Phase 1 聊到的信息来源，根据他的回答定制 400_Source 子目录：

```
读书多         →  401_读书笔记
追课多         →  402_课程笔记
都有           →  401_读书笔记 / 402_课程笔记 / 403_网络文章与归档
写个人长文多   →  + 404_个人笔记与随想
常跟 AI 聊     →  + 405_AI辅助产出
```

回头看 Phase 1——他有没有记日记/随记的习惯？
- 有 → 在 `100_Calendar/` 下再建一个 `101_随记/`
- 没有或不确定 → 跳过

#### 第 3 步：写 Obsidian 配置

写入 `.obsidian/` 下的配置文件：

**`.obsidian/app.json`**：
```json
{
  "attachmentFolderPath": "300_Extra/301_附件",
  "newFileLocation": "folder",
  "newFileFolderPath": "000_inbox"
}
```

**`.obsidian/core-plugins.json`**：确保 `zk-prefixer` 和 `templates` 为 `true`

**`.obsidian/zk-prefixer.json`**：
```json
{
  "format": "YYYYMMDD_HHmm_",
  "folder": "000_inbox",
  "template": "300_Extra/302_模版/卡片模板"
}
```

**`.obsidian/templates.json`**：
```json
{
  "folder": "300_Extra/302_模版"
}
```

> **[AI]** 配置已写好，你打开 Obsidian 就能直接用
> **[你]** 重启 Obsidian 让配置生效

#### 第 4 步：确认架构搭建完成

> **[AI]** 目录和配置全部就绪
> **[你]** 告诉我下一步——整理旧笔记还是直接开始学？

### Phase 4 — 迁移旧笔记（核心，共 7 步）

**前置**：目录和配置已搭好（Phase 3 完成），000_inbox/ 目录可用。

如果用户没有旧笔记或不想整理，**直接跳过本阶段**。

---

#### 第 1 步：准备旧笔记

> **[AI]** 帮你把旧笔记分类整理进卡片盒结构
> **[你]** 把旧笔记文件复制到 000_inbox/ 目录

引导用户复制旧笔记到 `000_inbox/`。支持几种回答路径：
- 用户自述路径（"在 D:/我的笔记/" ）→ 告诉他复制到 `000_inbox/` 的操作方法
- "已经放进去了" → 直接下一步
- "先不弄了" → 跳过本阶段

---

#### 第 2 步：全量扫描

> **[AI]** 读取 000_inbox/ 下所有文件的文件名和前几行内容
> **[你]** 等待扫描完成

AI 只读文件名和前 5 行，不读完整文件，节省时间。

---

#### 第 3 步：按角色归类

> **[AI]** 按 `references/classification.md` 的 6 种角色给每个笔记归类
> **[你]** 等待归类完成

判断按优先级从高到低走，命中即停。

| 角色 | 目标目录 | 说明 |
|------|---------|------|
| 外部输入类（书/课/文章） | 400_Source/ 对应子目录 | 摘录、书评、课程笔记 |
| 个人创作类 | 400_Source/404_个人笔记与随想 | 第一人称长篇叙事 |
| AI 对话类 | 400_Source/405_AI辅助产出 | 问答结构的对话记录 |
| 时间线记录类 | 100_Calendar/ | 日记、周记、工作日志 |
| 已提炼观点类 | 标记为「制卡候选」，留在 000_inbox/ | 原子化观点，可随时转为卡片 |
| 无法判断 | 留在 000_inbox/ | 特征不明显，用户自己决定 |

#### 第 4 步：生成分析报告

> **[AI]** 写入 `000_inbox/旧笔记分类建议.md`
> **[你]** 阅读报告内容

格式示例：

```
## 扫描结果：共 {N} 个文件

### 建议移入 401_读书笔记：
  - 《xxx》读书笔记.md
  - xxx摘录.md

### 建议移入 100_Calendar/：
  - 2024日记.md

### 制卡候选（留在 000_inbox/）：
  - 关于xxx的想法.md

### 待处理（留在 000_inbox/）：
  - 不知道放哪的文件.md
```

各子目录名以 Phase 3 实际创建为准。

#### 第 5 步：用户确认迁移方案

> **[AI]** 展示分类结果

> **[AI]** 展示迁移后的目录结构
> **[你]** 确认结果

告知用户：**"所有旧笔记已经按照卡片盒逻辑归类好了。接下来随时可以把 000_inbox/ 里的制卡候选素材提炼成卡片。"**

### Phase 5 — 教学：语法、模板、命名（共 5 步）

#### 第 1 步：学 Markdown 语法

> **[AI]** 教你最常用的几个 Markdown 写法
> **[你]** 打开 Obsidian 实际操作一遍

| 效果 | 写法 |
|------|------|
| 标题 | `# 一级` / `## 二级` / `### 三级` |
| 加粗 | `**文字**` |
| 列表 | `- 项目` |
| 分隔线 | `---` |
| 双链 | `[[卡片名]]` |
| 网页链接 | `[文字](URL)` |
| 嵌入图片 | `![[图片名.png]]` |

重点教 `[[` 双链——打完 `[[` 会弹出补全列表，选一张已有的卡就能连上，这个操作以后天天用。

教完之后说：**"够用了。就这几个，多了记不住也没必要。用多了自然熟。"**

#### 第 2 步：写入卡片模板

用 `Read` 读取 `assets/templates/卡片模板.md` 的内容，写入 `300_Extra/302_模版/卡片模板.md`。

各部分的含义见 `references/templates-guide.md`，教学时引用即可。

#### 第 3 步：写入 HQ&A 模板

用 `Read` 读取 `assets/templates/HQ&A模板.md` 的内容，写入 `300_Extra/302_模版/HQ&A模板.md`。

各部分的含义见 `references/templates-guide.md`，教学时引用即可。

#### 第 4 步：讲命名规则

命名规则见 `references/naming-rules.md`，教学时引用即可。

#### 第 5 步：展示一张真实的卡片

如果你自己有卡片，挑一张通俗易懂的展示给用户看。"看，这就是一张完整的卡，你也照这样来。"

### Phase 6 — 选做：试着制一张卡（共 4 步，可选）

> **[你]** 你有两个选择：
> - **想做** → 我带你从筛选到框架到出卡，走一遍完整流程
> - **跳过** → 没问题，直接去收尾，以后有空再做

#### 第 1 步：先教他筛选

筛选标准见 `references/card-filter.md`，据此引导用户判断。给他一个生活中常见的例子演示"两问"的用法，用"值得制"和"不值得制"各举例说明。AI 不要直接给答案，引导他自己分析。

#### 第 2 步：框定方向

确定值得做之后，AI 给出框架（但不写正文）：

```
建议标题：为什么周一总比周五觉得累？

核心观点：周一犯困不完全是懒，跟睡眠节奏被打乱、决策能量被周末消耗都有关系。

→ 建议链接方向
← [[相关文章标题]]        — 以后读到的相关内容可以链过来
→ [[意志力是有限资源]]    — 跟这张卡逻辑上连着
→ [[习惯是怎么建立的]]    — 也是同一类话题

延伸思考：
· 如果周一不安排重要会议，是不是更合理的工作方式？
· 有没有什么办法让周一的"启动成本"降低？

正文（你来写）：
```

#### 第 3 步：他自己写正文

> "好了，框架在这。去 Obsidian 里用卡片模板新建一张卡，把标题和框架填进去，正文写你对这个观点的理解，结合你自己的体会。写完之后把内容贴给我看，我帮你看看。"

#### 第 4 步：写完后给反馈

### Phase 7 — 收尾（共 3 步）

#### 第 1 步：生成入门总结

写入 `000_inbox/入门总结.md`，包含个人化上手清单：

```
□ 知道了卡片盒三大原则：一张卡一个观点 / 不瞎分类 / 靠链接串
□ 目录搭好了，Obsidian 也配好了
□ 旧笔记已整理到对应目录（如适用）
□ 会写 HQ&A 了，读完东西知道怎么记
□ 知道制卡前得用"两问"筛一遍
□ 知道了 ←（来源）和 →（延伸）怎么用
□ 理解链接是双向的——你引用了某张卡，那张卡也得能引用回来
□ 还记得最开始说的目标吗？这套系统能帮你实现它了
```

#### 第 2 步：生成架构文档

写入 `000_inbox/架构文档.md`。内容包括：
- 项目概览
- 目录结构及用途
- 卡片命名规范
- 模板清单
- 工作流说明
- 核心约定备忘
- 内容素材分析（扫描 000_inbox/ 和 400_Source/ 中现有素材）
- 后续推荐方向

格式参考 `references/architecture-template.md`，内容根据实际创建的 vault 目录和已有素材动态生成。

#### 第 3 步：进阶指路

> "现在你已经有了自己的卡片盒。接下来你的任务就是：往里面填卡。记住不是每个念头都值得做成卡——过两问，过不了就放。
>
> 等攒到 20-30 张卡的时候，你可以试试这几件事，帮你更高效地用这套系统：
>
> - **从已有笔记里提炼新卡**：翻翻你的 400_Source/，看看哪些内容值得做成原子卡片
> - **定期检查卡片链接**：看看哪些卡之间还没连起来，补上该有的双向链接
> - **深挖某张卡**：选一张觉得没想透的卡，问自己"这个观点的前提是什么？有没有反例？"
> - **让卡片互相碰撞**：找两张观点相反的卡放在一起，看看能碰撞出什么新想法
> - **基于卡片写文章**：卡片攒够了，挑一个主题串起来写一篇完整的文章
>
> 但现在不用想那么多——先记。先让卡片盒里有东西。祝你玩得开心。"

## 禁止事项（不得违反）

- **绝不替用户写卡片正文**。框架、标题、链接建议可以给，正文留白让用户自己写。
- **不对卡片内容做价值判断**。"这个观点对不对""有没有逻辑漏洞"——这是用户自己的事。
- **不代替用户思考**。"我觉得你可以写……"是错的。应该问"你想写什么？"
- **不改用户已有内容**。用户已有的卡片、笔记、文件，AI 只读不写。
- **不批量生成卡片**。卡片必须一张一张过"两问"筛选，不能一次性批量造。

如果用户的要求违反上述任意一条，拒绝执行并说明"这是卡片盒的必要难度原则决定的"。
