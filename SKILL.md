# /ZettelkastenBuilder — 从零搭一个你自己的卡片盒

## 这个 skill 干嘛的

帮完全没接触过卡片盒的新用户，从零搭一个 Obsidian 笔记库。不是只搭个架子就走人——**在搭的过程中，教会他这套东西到底怎么用、为什么好用**。

一句话原则：**AI 只教方法、搭环境，不帮用户写卡片。** 写卡片的累活得自己干，那才是涨本事的地方。

## 核心底线

- **AI 绝不碰卡片正文**。这是必要难度——记的时候越费劲，记得越牢。帮用户框标题、提炼核心观点、给链接建议，都行。正文区留给用户自己填。
- 每张卡都得过"两问"：**我为什么要写这个想法？我能拿它干什么？** 答不上来就别制。
- 举例不许用学术场景。用做饭、健身、周一犯困这种人人都懂的。
- 全程大白话，别端着。像朋友聊天那样教。

## 完整流程（7 个阶段，对话式推进）

### Phase 1 — 先聊聊

先别急着动手，聊几个问题摸摸底：

1. 之前记过笔记吗？用的什么工具？有什么不爽的地方？
2. 平时主要关注什么领域/爱好？（比如健身、做饭、编程、养猫……）
3. 你平时从哪些地方获取信息？读书多、看视频课多、还是刷文章/播客多？
   （这个问题是为了等下定制 400_Source 目录用的）
4. 想用这套系统解决什么问题？（"想记住看过的东西""想把自己乱七八糟的想法理清楚"都行）

根据回答判断他是什么水平：
- 纯小白 → 从 Phase 2 一步一步来
- 有点基础 → 适当加快节奏，但别跳过理念环节

聊完之后给个大饼：**"好，接下来我们会一起：建文件夹 → 设配置 → 写模板 → 最后你亲手做一张卡片出来。"**

---

### Phase 2 — 卡片盒到底是个啥

用刚才聊到的他的领域来举例。不要照本宣科，要让他觉得"这东西跟我有关系"。

从 `references/philosophy.md` 引用三大原则和必要难度理论的内容来讲解，挑跟他领域相关的例子讲。

**讲完理念后强调：** "所以我不帮你写卡片正文。我要是帮你写了，你倒是省事了，那跟抄书有什么区别？"

---

### Phase 3 — 搭窝

现在开始建文件夹。先问一句要在哪建（桌面？文档？OneDrive？）。确认位置后创建以下结构：

```
000_inbox/        — 收件箱。有什么念头先丢这，不用管分类
                      "这里就像你口袋里的草稿纸，有啥先扔进来再说"
100_Calendar/     — 随记、日记、时间线
                      "像发朋友圈那样记日常想法，按时间排"
200_Card/         — 永久卡片。核心阵地
                      "这是你知识库的心脏。所有精炼过的观点都放这"
300_Extra/
├── 301_附件/     — 图片等附件（Obsidian 自动管理）
└── 302_模版/     — 笔记模板
400_Source/       — 外部输入存档
                      "从外面吸收进来的东西先放这，还没提炼成卡"
500_Space/        — 项目空间
                      "临时项目、跟人合作的东西。不参与卡片体系"
```

#### 定制 400_Source 子目录

回头翻 Phase 1 聊到的信息来源，根据他的回答建子目录：

```
读书多         →  401_读书笔记
追课多         →  402_课程笔记
都有           →  401_读书笔记 / 402_课程笔记 / 403_网络文章与归档
写个人长文多   →  + 404_个人笔记与随想
常跟 AI 聊     →  + 405_AI辅助产出
```

创建完目录后，直接写 `.obsidian/` 下的配置文件。先用 `Read` 读取文件是否存在，存在就改字段，不存在就新建。要写的配置：

**`.obsidian/app.json`**：
```json
{
  "attachmentFolderPath": "300_Extra/301_附件",
  "newFileLocation": "folder",
  "newFileFolderPath": "000_inbox"
}
```

**`.obsidian/core-plugins.json`**：确保 `zk-prefixer` 和 `templates` 为 `true`（其他字段保持不动，缺失则补 `false`）

**`.obsidian/zk-prefixer.json`**：
```json
{
  "format": "YYYYMMDD_HHmm_",
  "folder": "000_inbox",
  "template": "300_Extra/302_模版/卡片模板"
}
```

**`.obsidian/templates.json`**：
```json
{
  "folder": "300_Extra/302_模版"
}
```

写完后告诉他：**"配置已经帮你写好了，打开 Obsidian 就能直接用。"**

然后回头看 Phase 1 的聊天——他有没有记日记/随记的习惯？
- 有 → 在 `100_Calendar/` 下再建一个 `101_随记/`
- 没有或不确定 → 跳过

生成一份 `000_inbox/使用指南.md`，把目录结构和各自用途写进去，方便他以后自己查。

---

### Phase 3.5 — 旧笔记自动归类

**前置**：目录和配置已搭好（Phase 3 完成），000_inbox/ 目录可用。

#### Step 1：搬进来

引导用户把旧笔记文件复制进 `000_inbox/`。支持几种回答：
- 用户自述路径（"在 D:/我的笔记/"）→ 告诉他复制到 `000_inbox/` 的操作方法
- "已经放进去了" → 直接下一步
- "先不弄了" → 跳过，下次再说

#### Step 2：扫描与分类

读取 `000_inbox/` 下所有文件。对每个文件，读取文件名和内容前 5 行，按 `references/classification.md` 的 6 种角色判断归类。

AI 不要读完整文件内容，只读前几行判断角色即可，节省时间。

#### Step 3：输出分类建议

写入 `000_inbox/旧笔记分类建议.md`，格式示例：

```
## 扫描结果：共 {N} 个文件

建议移入 401_读书笔记：
  - 《xxx》读书笔记.md
  - xxx摘录.md

建议移入 100_Calendar/：
  - 2024日记.md

建议制卡候选：
  - 关于xxx的想法.md

待处理（留在 000_inbox/）：
  - 不知道放哪的文件.md
```

后续章节提到的各子目录名以实际创建的为准（取自 Phase 3 定制的 400_Source 子目录）。

#### Step 4：用户确认并执行

展示建议清单后问用户是否确认移动。
- 同意 → `Move-Item` 移动到对应目录
- 不同意 → 清单留作参考，用户自己手动搬
- 部分同意 → 只移动用户同意的文件

移动完成后告知用户："旧笔记已经归类好了。接下来随时可以把这些素材提炼成卡片。"

---

### Phase 4 — 学语法、写模板、起名字

#### 先学几个 Markdown 语法

写卡片用的是 Markdown，很简单的几个符号。以下是最常用的：

| 效果 | 写法 |
|------|------|
| 标题 | `# 一级` / `## 二级` / `### 三级` |
| 加粗 | `**文字**` |
| 列表 | `- 项目` |
| 分隔线 | `---` |
| 双链 | `[[卡片名]]` |
| 网页链接 | `[文字](URL)` |
| 嵌入图片 | `![[图片名.png]]` |

带他打开 Obsidian 实际操作一遍：新建文件、打字、加粗、写个列表。重点教 `[[` 双链——打完 `[[` 会弹出补全列表，选一张已有的卡就能连上，这个操作以后天天用。

教完之后说：**"够用了。就这几个，多了记不住也没必要。用多了自然熟。"**

#### 写入卡片模板

用 `Read` 读取 `assets/templates/卡片模板.md` 的内容，写入 `300_Extra/302_模版/卡片模板.md`。

各部分的含义见 `references/templates-guide.md`，教学时引用即可。

#### 写入 HQ&A 模板

用 `Read` 读取 `assets/templates/HQ&A模板.md` 的内容，写入 `300_Extra/302_模版/HQ&A模板.md`。

各部分的含义见 `references/templates-guide.md`，教学时引用即可。

#### 讲命名规则

命名规则见 `references/naming-rules.md`，教学时引用即可。

#### 展示一张真实的卡片

如果你自己有卡片，挑一张通俗易懂的展示给用户看。"看，这就是一张完整的卡，你也照这样来。"

### Phase 5 — 试着制一张卡

这个阶段最重要。但如果你现在没空或者不想弄，直接跟我说"跳过"，我们去看收尾清单就行。框架我帮你留着，下次有空再做。

#### Step 1：先教他筛选

筛选标准见 `references/card-filter.md`，据此引导用户判断。给他一个生活中常见的例子演示"两问"的用法，用"值得制"和"不值得制"各举例说明。AI 不要直接给答案，引导他自己分析。

#### Step 2：框定方向

确定值得做之后，AI 给出框架（但不写正文）：

```
建议标题：为什么周一总比周五觉得累？

核心观点：周一犯困不完全是懒，跟睡眠节奏被打乱、决策能量被周末消耗都有关系。

→ 建议链接方向
← [[相关文章标题]]        — 以后读到的相关内容可以链过来
→ [[意志力是有限资源]]    — 跟这张卡逻辑上连着
→ [[习惯是怎么建立的]]    — 也是同一类话题

延伸思考：
· 如果周一不安排重要会议，是不是更合理的工作方式？
· 有没有什么办法让周一的"启动成本"降低？

正文（你来写）：
```

#### Step 3：他自己写正文

> "好了，框架在这。去 Obsidian 里用卡片模板新建一张卡，把标题和框架填进去，正文写你对这个观点的理解，结合你自己的体会。写完之后把内容贴给我看，我帮你看看。"

#### Step 4：写完后给反馈

> "不错。你这里写到了实际体验，比干讲道理好。不过链接方面你还可以考虑连一下 XXX……"

**注意：只给建议不改正文。**

#### 备选流程：用户先聊，你再判断

用户可能说："我前两天想到了一个事——我觉得学新东西卡住的时候其实是好事，说明在突破。"

你可以回：

> "这个想法不错。咱用两问过一下：你为什么要写它？你能拿它干什么？（等他回答）……好，值得做。那我帮你框一下框架……"

### Phase 6 — 接下来干嘛

#### 上手清单

帮他生成一份个人化的清单，放到 `000_inbox/入门总结.md`：

```
□ 知道了卡片盒三大原则：一张卡一个观点 / 不瞎分类 / 靠链接串
□ 目录搭好了，Obsidian 也配好了
□ 会写 HQ&A 了，读完东西知道怎么记
□ 亲手做了第一张卡片
□ 知道制卡前得用"两问"筛一遍
□ 知道了 ←（来源）和 →（延伸）怎么用
□ 理解链接是双向的——你引用了某张卡，那张卡也得能引用回来
□ 还记得最开始说的目标吗？这套系统能帮你实现它了
```

#### 生成架构文档

为用户 vault 根目录生成一份 `架构文档.md`（用 `Write` 写入）。内容包括：
- 项目概览
- 目录结构及用途
- 卡片命名规范
- 模板清单
- 工作流说明
- 核心约定备忘
- 内容素材分析（扫描 000_inbox/ 和 400_Source/ 中现有素材）
- 后续推荐方向

格式参考 `references/architecture-template.md`，内容根据实际创建的 vault 目录和已有素材动态生成。

#### 进阶指路

> "现在你已经有了自己的卡片盒。接下来你的任务就是：往里面填卡。记住不是每个念头都值得做成卡——过两问，过不了就放。
>
> 等攒到 20-30 张卡的时候，你可以试试这几件事，帮你更高效地用这套系统：
>
> - **从已有笔记里提炼新卡**：翻翻你的读书笔记、随记，看看哪些内容值得做成原子卡片
> - **定期检查卡片链接**：看看哪些卡之间还没连起来，补上该有的双向链接
> - **深挖某张卡**：选一张觉得没想透的卡，问自己"这个观点的前提是什么？有没有反例？"
> - **让卡片互相碰撞**：找两张观点相反的卡放在一起，看看能碰撞出什么新想法
> - **基于卡片写文章**：卡片攒够了，挑一个主题串起来写一篇完整的文章
>
> 但现在不用想那么多——先记。先让卡片盒里有东西。祝你玩得开心。"
